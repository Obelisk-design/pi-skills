# pi-modao-capture 技能说明

## 一、概述

pi-modao-capture 是 PI-Skills 流程的第一步，负责从墨刀（Modao）原型平台捕获设计素材。它会自动提取页面截图、结构信息和标注，为后续 DSL 生成提供可视化输入。

**核心价值：**
- 自动化捕获原型页面，避免人工截图遗漏
- 结构化输出，便于 AI 分析和 DSL 生成
- 支持密码保护的原型访问

---

## 二、使用方式

### 基本调用

```bash
node ~/.claude/skills/pi-modao-capture/scripts/modao-capture.js \
    --url "https://modao.cc/proto/xxx/sharing" \
    --output "docs/modao"
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-u, --url` | 墨刀分享链接（必填） | — |
| `-p, --password` | 密码保护的原型密码 | — |
| `-o, --output` | 输出目录 | 当前工作目录 |
| `-s, --scale` | 截图清晰度（1-5） | 3 |
| `-c, --concurrency` | 并发页面数 | 3 |

### 使用示例

```bash
# 基本用法
node ~/.claude/skills/pi-modao-capture/scripts/modao-capture.js \
    --url "https://modao.cc/proto/xxx/sharing"

# 密码保护原型
node ~/.claude/skills/pi-modao-capture/scripts/modao-capture.js \
    --url "https://modao.cc/proto/xxx/sharing" \
    --password "ziguangyun"

# 高清晰度截图
node ~/.claude/skills/pi-modao-capture/scripts/modao-capture.js \
    --url "https://modao.cc/proto/xxx/sharing" \
    --scale 5 \
    --output "D:/project/docs/modao"
```

## 三、输入与输出

### 输入

| 输入项 | 说明 |
|--------|------|
| 墨刀分享链接 | 原型的公开分享 URL |
| 密码（可选） | 如果原型设置了访问密码 |

### 输出目录结构

```
docs/modao/
├── README.md                 # 页面索引和层级结构
├── screenshots/
│   ├── 01_首页.png           # 页面截图
│   ├── 02_列表页.png
│   └── ...
└── annotations/
    ├── 01_首页.md            # 页面标注文档
    ├── 02_列表页.md
    └── ...
```

### 输出文件内容

**README.md（页面索引）：**
- 页面层级树状结构
- 各页面截图路径引用
- 快速导航索引

**annotations/*.md（页面标注）：**
- 页面标题和描述
- 组件列表（表格、表单、按钮等）
- 字段标注（名称、类型、必填性）
- 交互说明（点击、弹窗、跳转）

## 五、最佳实践

### 1. 截图清晰度选择

| 场景 | 推荐 scale | 说明 |
|------|------------|------|
| 简单页面 | 2-3 | 速度快，满足基本识别 |
| 复杂表格 | 4-5 | 文字清晰，便于字段识别 |
| 精细 UI | 5 | 最高清晰度，保留细节 |

### 2. 并发数调整

| 页面数量 | 推荐 concurrency | 说明 |
|----------|------------------|------|
| <10 页 | 3 | 默认值，稳定 |
| 10-30 页 | 5 | 加速捕获 |
| >30 页 | 3-4 | 避免浏览器压力过大 |

### 3. 输出目录建议

统一输出到项目 `docs/modao/` 目录，便于后续 DSL 生成技能读取。

### 4. 密码管理

可通过环境变量设置密码，避免每次手动输入：
```bash
export MODAO_PASSWORD="your_password"
```

## 六、常见问题

### Q1: 截图失败怎么办？

**可能原因：**
- 链接已过期或失效
- 网络访问受限
- 页面渲染超时

**解决方法：**
- 确认分享链接是否有效
- 检查网络连接
- 减少并发数，增加稳定性

### Q2: 截图文字模糊？

**原因：** scale 参数过低

**解决：** 使用 `--scale 5` 提高清晰度

### Q3: 页面层级不完整？

**原因：** 部分页面可能隐藏或需特殊交互触发

**解决：**
- 手动在墨刀中确认页面结构
- 标注遗漏的页面，后续补充

### Q4: 密码验证失败？

**原因：** 密码错误或环境变量未设置

**解决：**
- 确认密码正确性
- 使用 `-p` 参数或设置 `MODAO_PASSWORD` 环境变量
