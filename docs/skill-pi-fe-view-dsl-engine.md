# pi-fe-view-dsl-engine 技能说明

## 一、概述

pi-fe-view-dsl-engine 是前端 DSL 生成技能，将墨刀原型截图和标注转化为结构化的前端组件 DSL。这个 DSL 定义了页面布局、组件树、字段结构、交互规则，是前端代码生成的核心规格文档。

**核心价值：**
- 结构化记录设计意图，避免口头理解偏差
- 自动推断数据绑定关系，明确 API 消费点
- 提供 Layer 2 审查的 DSL 基准

---

## 二、使用方式

### 触发条件

用户触发 `/pi-fe-view-dsl-engine` 命令，或说：
- "从原型生成页面结构"
- "把墨刀截图转成组件树"
- "生成前端 DSL"

### 前置条件

必须先完成 `pi-modao-capture`，确保 `docs/modao/` 目录存在截图和标注。

### 输入读取

AI 会自动读取：
- `docs/modao/screenshots/*.png` — 页面截图
- `docs/modao/annotations/*.md` — 页面标注
- `docs/modao/index.md` — 页面层级

## 三、输入与输出

### 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 页面截图 | docs/modao/screenshots/ | 可视化布局参考 |
| 页面标注 | docs/modao/annotations/ | 组件、字段、交互说明 |
| 页面层级 | docs/modao/index.md | 页面结构关系 |

### 输出目录结构

```
docs/dsl/frontend/
├── index.md                          # DSL 索引
└── {页面名}.dsl                       # 每个页面的 DSL 定义
```

## 四、DSL 语法结构

### 页面根块

```dsl
page "ComponentName" {
  title: "显示标题"
  layout: standard-page
  
  // 组件块...
}
```

### 布局类型

| 类型 | 说明 |
|------|------|
| `standard-page` | 标准页面（面包屑、工具栏、内容区） |
| `modal` | 弹窗对话框 |
| `drawer` | 侧边抽屉 |
| `card-section` | 卡片区块 |
| `tab-panel` | 标签面板 |

### 组件块示例

```dsl
table "SpaceTable" {
  dataSource: "${api.space.list}"
  columns: [
    { key: "id", title: "ID", width: 80 },
    { key: "spaceName", title: "空间名称", sortable: true },
    { key: "status", title: "状态", type: "tag" }
  ]
  actions: [
    { key: "edit", label: "编辑", action: "openForm" },
    { key: "delete", label: "删除", action: "confirmDelete" }
  ]
}

modal "SpaceForm" {
  title: "${isNew ? '新增' : '编辑'}"
  fields: [
    { key: "spaceName", type: "input", label: "空间名称", required: true },
    { key: "spaceCode", type: "input", label: "空间编码", required: true }
  ]
  actions: ["submit", "cancel"]
}
```

### 字段类型

| 类型 | 说明 |
|------|------|
| `input` | 文本输入框 |
| `textarea` | 多行文本 |
| `select` | 下拉选择 |
| `date` | 日期选择器 |
| `dateRange` | 日期范围 |
| `number` | 数字输入 |
| `switch` | 开关 |
| `upload` | 文件上传 |
| `cascader` | 级联选择 |

### 数据绑定语法

使用 `${...}` 占位符标记数据来源：

| 语法 | 说明 |
|------|------|
| `${api.space.list}` | API 端点引用 |
| `${isNew ? 'A' : 'B'}` | 条件表达式 |
| `${form.spaceName}` | 表单字段引用 |
| `${row.status}` | 表格行字段 |

## 六、最佳实践

### 1. 组件命名

使用 PascalCase 命名，反映组件用途：
- `SpaceTable`（空间列表表格）
- `SpaceForm`（空间表单）
- `UserFilterPanel`（用户筛选面板）

### 2. 字段完整性

保留标注中的所有字段，不主动删减。遗漏字段由后续 Review 发现。

### 3. 数据绑定占位符

使用 `${api.resource.action}` 格式，不硬编码真实 API 路径。由 `pi-be-data-dsl-engine` 解析。

### 4. 嵌套组件处理

弹窗、抽屉作为页面 DSL 的嵌套块，不单独成文件。

### 5. 人工确认点

以下情况需人工确认：
- 截图文字模糊无法识别
- 标注信息不完整
- 组件类型推断存疑

## 七、常见问题

### Q1: 截图识别错误怎么办？

**原因：** 截图模糊、文字不清

**解决：**
- 使用 `pi-modao-capture --scale 5` 提高清晰度
- 人工标注补充遗漏字段

### Q2: 字段命名与 API 不一致？

**原因：** DSL 推断的命名与 YApi 定义不同

**解决：**
- Layer 1 审查会发现命名不一致
- 修正 DSL 或确认 API 字段名

### Q3: 级联关系遗漏？

**原因：** 截图未显示级联逻辑

**解决：**
- 人工标注级联关系
- 补充 `dependentOn` 属性

### Q4: 弹窗内容未捕获？

**原因：** 弹窗需点击触发，截图未显示

**解决：**
- 手动触发弹窗后截图补充
- 或人工标注弹窗内容
