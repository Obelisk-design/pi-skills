# PI-Skills 工作流简介

## 概述

PI-Skills 是一套用于前端开发全流程管理的 AI 辅助技能集合，覆盖从设计原型捕获到代码审查的完整开发链路。

**核心目标：**
- 将设计原型转化为结构化的 DSL（领域特定语言）
- 自动生成符合设计规范和 API 约定的前端代码
- 三层验证确保代码质量

---

## 各 Skill 功能说明

### 1. pi-modao-capture（设计捕获）

**功能：** 从墨刀原型分享链接提取页面截图和标注

**输入：** 墨刀原型分享链接（可能需要密码）

**输出：**
- `docs/modao/screenshots/*.png` — 页面截图
- `docs/modao/*.md` — 页面标注文档
- `docs/modao/index.md` — 页面层级索引

**使用场景：** 项目开始时，将设计师的墨刀原型转为可分析的素材

---

### 2. pi-yapi（接口文档导出）

**功能：** 从 YApi 平台导出接口文档到本地

**输入：** YApi 项目/分类 ID（如 `http://10.0.53.135/project/3116/interface/api/cat_32621`）

**输出：**
- `docs/yapi/{项目名}/README.md` — 接口索引
- `docs/yapi/{项目名}/{ID}_{标题}.md` — 每个接口详情

**使用场景：** 导出 API 定义，用于后续 DSL 生成和代码审查

---

### 3. pi-fe-view-dsl-engine（前端视图 DSL）

**功能：** 将墨刀截图转化为前端组件树 DSL

**输入：** `docs/modao/` 目录下的截图和标注

**输出：**
- `docs/dsl/frontend/{页面}.dsl` — 每个页面的组件结构
- `docs/dsl/frontend/index.md` — DSL 索引

**DSL 内容包括：**
- 页面布局结构
- 组件类型（表格、表单、弹窗等）
- 字段定义（类型、必填、验证规则）
- 交互动作（按钮点击、表单提交）
- 数据绑定占位符

**使用场景：** 为代码生成提供页面结构蓝图

---

### 4. pi-be-data-dsl-engine（后端数据 DSL）

**功能：** 将 YApi 接口转化为数据转换 DSL

**输入：** `docs/yapi/{项目名}/` 目录下的接口文档

**输出：**
- `docs/dsl/backend/{接口}.dsl` — 每个接口的数据定义
- `docs/dsl/backend/README.md` — DSL 索引

**DSL 内容包括：**
- 请求参数校验规则
- 响应数据类型定义
- 字段映射和转换规则
- 枚举值定义

**使用场景：** 定义 API 数据契约，供前端 DSL 绑定

---

### 5. pi-integrate-fe-be（前后端 DSL 集成）

**功能：** 合成前端视图 DSL 和后端数据 DSL，生成全栈集成 DSL

**输入：**
- `docs/dsl/frontend/` — 前端组件树
- `docs/dsl/backend/` — 后端数据定义

**输出：**
- `docs/dsl/integrated/{页面}.dsl` — 全栈集成 DSL
- `docs/dsl/integrated/index.md` — 集成索引

**集成内容包括：**
- 组件绑定的 API endpoint
- 数据流向（选择 → 加载详情，提交 → 刷新列表）
- 未绑定的组件/API 标记

**使用场景：** 作为代码生成的完整规格说明

---

### 6. pi-review（代码审查）

**功能：** 三角交叉验证生成的代码是否符合设计规范和 API 约定

**输入：**
- 生成的代码文件
- `docs/yapi/` — API 合约
- `docs/dsl/frontend/` — DSL 定义
- `docs/modao/` — 设计原型

**输出：**
- `docs/review/review-report.md` — 审查报告

**三角交叉验证架构：**

```
        Modao截图（设计源）
           ↑
          / │
         /  │
  L3:   /    │ L2.5（新增）
 Code  /      │ DSL
 vs   /        │ vs
Modao          │ Modao
    ↓          ↓
   Code ←──── DSL（L2）
```

**四层验证优先级：**

| 层级 | 对比关系 | 优先级 | 来源 | 通过阈值 | 说明 |
|------|----------|--------|------|----------|------|
| Layer 1 | Code ↔ YApi | 阻塞级 | YApi | 100% | API合规：URL、方法、必填参数 |
| Layer 2 | Code ↔ DSL | 主要级 | DSL | ≥90% | DSL合规：组件、字段、数据绑定 |
| Layer 2.5 | DSL ↔ Modao | 主要级 | Modao | ≥90% | DSL准确性：检测DSL生成错误 |
| Layer 3 | Code ↔ Modao | 次要级 | Modao | ≥80% | UI保真：布局、标签、视觉层级 |

**交叉验证诊断：**

| L2结果 | L3结果 | L2.5结果 | 诊断 | 处理建议 |
|--------|--------|----------|------|----------|
| PASS | PASS | (跳过) | ✅ 全部正确 | 无需处理 |
| PASS | FAIL | FAIL | DSL生成错误 | 重新生成DSL |
| PASS | FAIL | PASS | 视觉实现错误 | 调整代码样式/布局 |
| FAIL | FAIL | PASS | 代码生成错误 | 修正代码匹配DSL |
| FAIL | PASS | FAIL | DSL错误但Code人工修正 | 同步DSL |

**核心价值：**
- 单链路对比可能遗漏"两者都错"的情况
- 三角验证通过交叉对比可以发现DSL或Code的独立错误
- Layer 2.5 专门检测DSL生成阶段的识别偏差

**自动化脚本：**
```bash
node ~/.claude/skills/pi-review/scripts/dsl-compare.js \
  --dsl-file docs/dsl/frontend/页面.dsl \
  --code-dir src/views/模块/ \
  --output docs/review/dsl-check.json
```

### 7. pi-workflow（工作流协调）

**功能：** 协调上述 skills 按顺序执行

**使用方式：**
- 用户说"启动工作流"、"执行完整流程"
- 自动按 Phase 1 → Phase 5 执行
- 每阶段完成后报告状态

## 快速开始

### 完整流程

1. **准备素材**
   - 墨刀原型分享链接
   - YApi 项目/分类链接

2. **执行 Phase 1**
   ```
   /pi-modao-capture --url "墨刀链接" --password "密码"
   /pi-yapi http://10.0.53.135/project/xxx/interface/api/cat_xxx
   ```

3. **执行 Phase 2-3**
   ```
   /pi-fe-view-dsl-engine
   /pi-be-data-dsl-engine
   /pi-integrate-fe-be
   ```

4. **代码生成**
   - 基于 `docs/dsl/integrated/` 手动或自动生成代码

5. **代码审查**
   ```
   /pi-review
   ```

### 单独使用

每个 skill 可独立调用：
- 只导 API 文档 → `/pi-yapi`
- 只捕获原型 → `/pi-modao-capture`
- 只审查代码 → `/pi-review`

---

## 注意事项

1. **Cookie 过期** — YApi cookie 有效期有限，过期时需重新获取
2. **原型密码** — 墨刀私有原型需要密码才能访问
3. **DSL 同步** — API 变更后需重新运行 DSL 生成
4. **审查阻塞** — Layer 1 不通过会阻止后续检查
